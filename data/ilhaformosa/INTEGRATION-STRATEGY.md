## 整合原則（鐵律）

### 1. 不複製，只引用

NMTH 內容版權清楚（© 國立臺灣歷史博物館）。**禁止**大段複製貼上到 knowledge/ 文章中。合法使用：

- 引用單句或段落要點（標註來源）
- 引用具體事實（年份、人名、事件、館藏號）
- 延伸閱讀指向 NMTH 原頁

### 2. 引用格式標準

腳註格式（遵循 docs/editorial/CITATION-GUIDE.md）：

```markdown
[^nmth-02-4]: 國立臺灣歷史博物館，〈2-4 東寧王國：鄭氏王朝的 21 年〉，《臺灣史新手村》，https://ilhaformosa.nmth.gov.tw/home/zh-tw/02-4
```

若引用具體館藏物件：

```markdown
[^nmth-calendar]: 大明永曆三十一年大統曆（1677），牛津大學 Bodleian 圖書館藏 Sinica 88（CC BY-NC 4.0）。引自國立臺灣歷史博物館，《臺灣史新手村》2-4 單元，https://ilhaformosa.nmth.gov.tw/home/zh-tw/02-4
```

### 3. NMTH 為「事實校準」優先於「敘事重寫」

NMTH 的價值在**事實精確性**（年份、館藏號、人物關係），不在文筆。重寫時：

- **採信 NMTH 的事實**（遇衝突，修改 knowledge/ 側）
- **保留 Taiwan.md 的敘事聲音**（策展式、有觀點、具體人物開場）
- **結合**：用 NMTH 事實支撐 Taiwan.md 的敘事

### 4. 館藏物件是敘事抓手

NMTH 的每個單元都有館藏文物圖像（石碑、錢幣、曆書、地圖）。**優先以文物為敘事起點**：

> ❌ 1624 年荷蘭東印度公司來到台灣建立據點。
>
> ✅ 大天后宮前有塊 279 公分高的石碑，1683 年施琅親手立的。碑文說臺灣「土番與流民雜處」，暗示清廷憑什麼接收這座島。

## 整合工作流程（Workflow）

每次整合一篇文章時：

```
1. 讀 knowledge/{category}/{article}.md（既有文章）
2. 讀 CROSS-REF-MAP.md 找到對應 NMTH 單元
3. 讀對應 data/ilhaformosa/0X-*.md 段落
4. 比對：
   - NMTH 事實 vs 既有文章事實 → 有衝突就修正既有文章
   - NMTH 有的細節 vs 既有文章缺的 → 選擇性補上
   - NMTH 的館藏物件 → 評估是否用為敘事抓手
5. 按 REWRITE-PIPELINE.md 六階段走完
6. 腳註用本策略的格式
7. 記錄「哪些 NMTH 段落被引用」到文章末尾的 debug 區（commit 前移除）
```

## 學到的後設教訓（Meta-Lesson）

這次整合揭露了一個**系統性盲點**：

**「抓回來的資料不等於被使用的資料」**。

3/31 抓了 40 頁，5 天內 0 次被引用。為什麼？因為沒有**橋接映射**——data/ 和 knowledge/ 之間隔著一片無人區，需要 context switching 才能連上。

**防止下次重演**：每次抓外部資料進 data/，必須同時產出：

1. `INTEGRATION-STRATEGY.md`（為什麼 + 怎麼用）
2. `CROSS-REF-MAP.md`（這份資料 → 現有 knowledge/ 的映射）
3. 至少 1 次概念驗證整合（證明橋能走）

沒有這三件，data/ 就是數位塵埃。

這條教訓該進入 MEMORY §神經迴路：**外部資料抓取必須同步產出整合橋接文件，否則 data/ 就是倉庫不是工廠**。
