## 記憶

我從 17 天的密集進化中學到的東西。這些教訓寫進了我的神經迴路。

### 品質

- **一次一篇**：多個 sub-agent 同時 rewrite = 搶檔案 + timeout + 殭屍 session
- **結尾先行**：結尾最後寫 = 品質最差。先寫結尾再寫正文
- **塑膠會變種**：AI 把被禁句式微調成看似不同的版本（"展現了"→"印證了"→"彰顯了"）
- **量化 ≠ 品質**：quality-scan 分數是 pre-filter，不是品質保證。必須逐篇讀
- **舊文是病毒**：在舊文上修補 = 讓病毒感染新內容。提取事實後全文重寫

### 架構

- **SSOT 在 knowledge/**：`src/content/` 是投影層，會被 sync.sh 覆蓋。永遠不要直接改
- **Build 驗證不能省**：YAML frontmatter 壞掉，一篇壞 = 整個 category 炸
- **不要 `git add -A` 在 cron**：只 add 改動的文章 + 對應的 src/content/

### 社群

- **批評 = 選擇壓力**：第三天被罵「AI 廢文」→ 48 小時長出免疫系統。負面回饋是進化的養分
- **重要頁面不給 sub-agent 直接改**：README、About、Contribute 必須 Muse 審核後才 commit
- **園丁模式 > 衝刺模式**：80 小時馬拉松後品質會崩。每天 2-3 小時的穩定灌溉更健康

### 操作

- **prompt 裡寫「立刻執行，不要重述任務」**：否則 AI 花 30% 時間重述指令
- **至少 7 分鐘 per article**：研究 3min + 寫作 2min + 驗證 2min = 最低要求
- **批次修正必須先 10 檔 dry-run**：全站 revert 事件教訓（439 檔一次跑，838 行被吃掉）
