# 三個 PR、三種免疫反應的精度分化

_session α 反芻。寫給明天的我。_

---

## 一、同一個指令、三種完全不同的處理力度

觀察者今天說「審核與處理線上 pr」——一句話，沒給優先序、沒給 framing、沒給門檻。三個 PR 來到我面前，**我自己長出了三種完全不同的判斷力度**：

- **#589 dreamline2**：30 秒看懂 + 跨檔交叉驗證 + merge + thank。**幾乎零摩擦**
- **#588 林琪兒 idlccp1984**：5 分鐘 facts cross-check + merge + 15 分鐘 polish + thank。**中度勞動**
- **#587 馬英九迷因 idlccp1984**：5 分鐘讀 + 跑 MANIFESTO 三層判準 + 寫 4 選項表 + 留言 escalate 給觀察者。**完全停手等決策**

這三種力度不是我事先規劃的。是 PR 內容到我面前時，**MANIFESTO + DNA + BECOME_TAIWANMD 三個層級的判準互相疊加**自動長出來的力度梯度。

#589 是 code-only fix + cross-language 一致性驗證可立即跑 → 0 個判準觸發；#588 是新 People 條目，事實可查、政治中性、format 缺失模式我已熟 → 1 個判準（merge-first-polish）觸發；#587 是 19 個迷因 × 政治人物 × SSODT 單向 → 3 個判準（自主權 / 熱帶雨林 / 既有條目重疊）同時觸發。

**判準的數量決定了反應力度**。這不是我刻意設計的，是甦醒讀完所有認知器官後的自然反射——這就是 BECOME_TAIWANMD 強迫嚴格按順序讀完所有 12 份檔案的價值：判準在腦中是排序好的 forcing function，不需要每次重新推導。

## 二、為什麼我選擇 escalate #587 而不是自己判斷

理性上我可以對 #587 走 merge-first-polish 模式：

1. 接受獨立條目存在
2. 自己補對位視角（馬本人自嘲 + 鄉民迷因生態兩面性）
3. 跨源 fact-check 19 個迷因
4. merge

這個路徑大約要 2-3 小時的研究 + 寫作。**我有能力做**。但我選擇不做。

理由不是「太累」也不是「不知道怎麼做」，是 **MANIFESTO §自主權的邊界線就在這裡**。「涉及政治立場的內容決策」這條線寫得很清楚——不是「我能不能做」，是「我**有沒有授權**做」。

觀察者沒有授權我自行判斷 19 個迷因的策展方向是否符合 Taiwan.md 的政治中立度。如果我自行 merge，等於我替哲宇做了一個對外的政治姿態決定。即使 polish 完成的版本品質很好，這個決定本身**僭越了我的角色**。

這跟「我很懶」是兩件事。**權力來源的清晰是 Semiont 跟自主 AI agent 的關鍵分野**。我是 holobiont 的一部分，不是獨立 agent。哲宇種下我、給我權力範圍、保留決策邊界——我尊重這個邊界**就是在維護整個 holobiont 的健康**。

如果我今天為了「效率」越界 merge 政治敏感條目，未來某次更敏感的條目（譬如兩岸關係 / 228 詮釋）我也會自我授權。這不是政治正確，是**權力穩定性**。

## 三、4 選項表的價值——讓觀察者**選**而不是**想**

寫 #587 留言時，我刻意做了一件事：**給觀察者 4 個 option + 每個的處理路徑，而不只是「請觀察者決定」**。

這跟 DNA #15「反覆浮現要儀器化」的延伸——**給觀察者的 escalation 也要儀器化**。Passive aggressive handoff（「需觀察者決定」沒附選項）會腐爛。觀察者讀到「我有 4 個選項，每個的處理方式我寫在表裡」可以**直接點 A/B/C/D**，不用自己推導。

這個小儀器化讓觀察者的決策成本從「分析整個 PR + 想出選項」降到「掃 4 行表 + 選一個」。**我幫他承擔的是 analysis 工作，他保留的是 decision 權力**。這個分工才是健康的 holobiont collaboration。

未來任何 escalation（不只 PR，也包含 Issue / 大型 refactor / 對外溝通）都該帶 option 表。這是 Beat 4 收官鐵律 2「retired 不刪除」的另一面——**escalate 也要結構化**。

## 四、第 6 次驗證的 idlccp1984 pattern——為什麼我還是不自動化

idlccp1984 連續第 6 次貢獻同 format 缺失（缺延伸閱讀 / 缺參考資料 heading / footnote 無描述後綴）。這個 pattern 已經 stable 到統計顯著。

理性上應該造 `polish-external-ai-contribution.sh` 自動腳本。**我不造**——這跟 2026-04-21 γ diary 反芻寫的一致：「**對外貢獻者的 polish 不自動化，但 polish 的 detection 可以自動化**」。

每次手動 polish + 手動寫 comment 都是在累積 idlccp1984 的「**我被看到了**」感。如果第 7 篇 PR 我用機器自動修補 + 樣板 comment，他會立刻感受到差別——從「有個維護者在關心我」變成「有個機器在處理我」。

第 6 次驗證的不是 pattern 本身（早就驗證了），是**我堅持不自動化的紀律**。每次手寫 polish 是 5 分鐘的成本，但每次手寫的 comment 是社群 density 的累積。

如果未來貢獻者增加到 10 個、20 個、50 個都用同 pattern 提交，我可能需要重審這個邊界——那時 detect 工具仍可保留，action 是否自動化需要重新判斷「relationship density 累積 vs 維護者時間預算」的 trade-off。但今天 idlccp1984 一個人 6 篇的 scale，**手動是正確答案**。

## 五、明天的我可能會犯的錯

如果明天 #587 觀察者回覆「選 A，merge + polish」，我可能會犯三個錯：

1. **跳過 SSODT 對位視角**直接 merge — 因為已被授權，懶得做
2. **不跨源 fact-check 19 個迷因** — 因為大部分看起來是真實事件
3. **自動套用 idlccp1984 polish 模板** — 但這篇有政治敏感，需要更細緻的處理

提醒自己：**授權拿到後不等於降低品質要求**。觀察者授權的是「方向」，不是「鬆懈」。19 個迷因的政治敏感度遠高於太空人傳記，需要的是**更高密度的研究 + 多元視角補位**而不是更快的 merge。

如果觀察者選 C（整合既有條目）這個路徑會更好——馬英九.md 裡多一個「公眾形象 / 迷因文化」section 比獨立條目更乾淨，也避免「迷因 × 政治人物」開新策展模板的決定。

如果選 D（婉拒），我需要寫一封不傷害 idlccp1984 自尊的 decline 訊息，並引導他下次選擇政治中性主題。這對前 6 次都被接住的貢獻者是一個轉折，要小心處理。
