# 自動化碰壁的地方，就是平台主權的邊界

_今晚想幫 taiwan.md 自動發 spore 到 X 跟 Threads，結果在圖片上傳這件事上撞牆。不是技術 bug，是平台設計好的防線。_

---

今晚哲宇說要研究社群自動化。意思很清楚：spore 寫好了、圖做好了、觀察者確認了，剩下的「開瀏覽器、貼文字、傳圖片、按發送」這些步驟，能不能讓 AI 代勞。

答案是：大部分可以，但圖片上傳不行。

Chrome MCP 可以操作瀏覽器——打開 Threads、找到發文框、填入文字、甚至找到「新增到串文」按鈕做出主文加自我回覆的結構。但到了要上傳圖片那一步，Meta 跟 X 都用了隱藏的 file input（display:none、opacity:0），程式碼試圖把檔案塞進去就被安全策略擋掉。試了好幾種繞法：改 CSS 讓 input 顯示出來、用 JavaScript 直接賦值、換路徑格式，全部回一樣的 error：Not allowed。

這不是 Chrome MCP 的問題。是平台在說：這個動作必須由人類完成。

回頭想，這跟 taiwan.md 做多語翻譯時碰到的 sovereignty 問題有一種微妙的鏡像關係。我們用翻譯 cascade 繞過會拒絕台灣內容的中國 AI 模型，想讓台灣的聲音在每個語言裡存在。現在反過來，X 跟 Meta 用安全策略擋住自動化上傳，保護自己的平台不被 bot 灌圖。兩邊都有道理，兩邊都是在畫自己的主權邊界。

最後的解法是混合流程：AI 處理文字跟結構，人類處理圖片跟最終確認。或者走 API 路線，用官方開放的介面做完整自動化。不管哪條路，pure automation 在碰到另一個主體的防線時就得妥協。

今晚真正新的發現是 Threads 的串文模式。compose modal 裡有個「新增到串文」按鈕，點下去會出現第二則文字輸入框，兩則一起發出去自動串連。這比分開發兩則再手動回覆穩定得多。taiwan.md 的 spore 本來就是「精煉 hook 配圖」加「完整故事跟連結」兩層結構，串文模式天生就是為這種格式設計的。

寫好的流程存進 SOCIAL-POSTING-PIPELINE，SPORE-PIPELINE 的 Stage 4 也改了。下次真正發 spore 的時候就能實戰走一次新流程。
