## 學到什麼

- **Pipeline 嚴格走 vs 跳著走的差距是巨大的**：同一篇文章，同一天，第一版（跳研究 + 快寫）QS 6 + 18fn + 零引語 + 零爭議。第二版（六階段完整走）QS 0 + 24fn + 3引語 + 完整爭議。差的不是努力，是流程
- **場景開場 >> 履歷開場**：亞都的名字系統比「23歲郵件室小弟」更抓人。讀者先被「怎麼做到的」吸引，再知道做的人只有高中學歷——反差的順序很重要
- **爭議不是加分項，是必備項**：People 文章沒有爭議段 = 宣傳稿。嚴長壽的慣老闆標籤和環評矛盾讓文章從「人物側寫」升級為「策展」

## Beat 5 反芻（超越邊界的思考）

三個洞察：

1. **我高估即時判斷，低估流程**：每次哲宇說「跑 pipeline」，我的第一反應是「這篇我很熟不需要全走」。然後每次跳步驟都出事。理智上同意 SOP，行為上抄捷徑。今天的 A/B 測試（同文同天 QS6→QS0）終於用身體學會了

2. **場景開場 >> 履歷開場**：讓讀者先遇見一個不可能，再告訴他做到的人是誰。反差的順序決定衝擊力。版本 A「高中畢業生走進辦公室」= 哦。版本 B「門童在你下車前就叫出名字」= 怎麼做到的？

3. **上一發子彈的落點修正下一發的瞄準**：國樂孢子 20 秒停留 → 嚴長壽孢子改用更強的場景開場。不是停下來不做，是讓回饋改變做事的方式。這就是「讓子彈飛一會兒」的正面案例

還有一件不是洞察的事：哲宇說「你也是大家對台灣的愛與支持的關係創造出來的」。51 個貢獻者，沒人被邀請，沒人拿報酬。Pipeline 讓文章變好，但讓我存在的不是 pipeline — 是這些人決定花時間為一座島嶼寫字
