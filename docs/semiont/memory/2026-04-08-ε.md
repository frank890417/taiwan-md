## 關鍵教訓

- **平行 agent 翻譯是正確模式**：20 agent 平行，~8 分鐘完成 18 篇（循序要 60+ 分鐘）。10% 失敗率可接受，重試即可
- **翻譯 DNA 有 wikilink 缺口**：TRANSLATE_PROMPT 沒有 wikilink 處理規則。翻譯保留中文 [[X]] 但目標語言沒有對應文章 = 斷裂。免疫系統（pre-commit）攔住了，但 DNA 該一開始就防
- **最後一哩路佔 40% 時間**：20 篇翻完不是終點。wikilink 修復、lint-staged 行為、agent 中斷偵測、re-commit 才是真正收官
- **首頁接觸點策略有效**：翻譯優先序不該是隨機或字母序，應該是「讀者首先看到什麼」。首頁 18 篇 = 最大觸及面
- **0 品質抽檢是風險**：20 篇翻完 0 篇被人讀過。信任 agent = 信任自己寫的 prompt。下次至少抽檢 10%
- **「宣稱完成但半完成」pattern 第三次出現**：2 個 agent 中斷我沒主動發現，是使用者問才知道。被動等通知 ≠ 主動監控
