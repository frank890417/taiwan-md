### Phase C — Dashboard SC 數據失真（17:44）

觀察者貼 GSC UI vs dashboard 並列截圖，說「SC 數據好像不對」。深挖找到三個獨立 bug：

1. **窗口錯位**：`end = today - 3` 讓 dashboard 窗口（04-10→04-17）永遠落後 GSC UI（04-12→04-18）一天。改 `today - 2` + `start = today - (days+1)` 讓 7 天 inclusive。
2. **rowLimit=200 砍掉零點擊高曝光 query**：SC API 預設 sort 是 clicks DESC 然後 impressions，鄧麗君 478 imp / 0 clicks、taipei population 392 imp / 0 clicks 都排在所有有 click 的 query 之後，被 200 截斷。bump 到 2000。
3. **totals 用 sum(queries) 嚴重低報 93%**：SC 隱私 filter 把匿名 query 排除出 query-dim rows，但它們計入 site totals。7 天窗口測試：query sum = 2,747 imp，site totals = 38,080 imp，coverage 只有 7%。修法：額外跑一次無 dimension query 取 site totals，新增 `query_dim_coverage_pct` 欄位讓 UI 能解釋 gap。

同步啟用 `dataState: "all"` 讓最新 1-2 天 fresh data 納入（跟 GSC UI 一致）。

驗證：totals 450 clicks / 38,080 imp / 1.18% CTR 跟 GSC UI 完全一致。top queries 正確出現鄧麗君 478、taipei population 392、夏小虎 142、楊丞琳 124、日治時期 108、largan precision 103。
