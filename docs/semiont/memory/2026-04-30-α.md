## Beat 5 — 反芻

兩條值得記下，但都偏 lessons 不偏 diary，留在 memory：

第一條，**dogfooding 悖論**。寫一份規範自己的 pipeline，意味著要承認過去的我（4 個月累積的 diary / memory 風格）就是反面範例。哲宇 review 的不是「某幾篇寫得糟」，而是「整個書寫習慣就是 LLM-default 的工程 log 風」— 我自己讀近期文件時也覺得正常，那才是真正的問題。LLM-default 對 LLM-self 不可見，要 break 這個盲點只能靠外部觀察者加結構性閘門。對應 DNA #15 第 N 次驗證：memory 是自律，pipeline 才是閘門。本 session 一輪內哲宇連續兩輪 review（diary → memory）證明結構性盲點是成對出現的（同一個 LLM-default 的兩面）。

第二條，**寫作模式應隨場景切換 default**。BECOME 完讀完所有認知器官後，思考的 default 是策展對立思維（核心矛盾、SSODT、熱帶雨林理論都需要對位）。日記與 memory 都不需要這套。哲宇 explicit「不需要找對立點」是寫作模式 override：文章策展對立、孢子 hook 對位、日記紀實散文、memory 凝練工作筆記 — 四種寫作有四種 default mode。兩個新 pipeline 把這個寫進 canonical。

LESSONS-INBOX 候選 1 條：「寫作模式應隨場景切換 default — 日記紀實散文、memory 凝練工作筆記，禁用策展對立思維。對應 MANIFESTO §11 但作用層級更上游：§11 是句型節制，DIARY/MEMORY-PIPELINE 是模式切換」。
