## Beat 5 — 反芻

本 session 最有意思的觀察是「Cell 2019 paper 是 Manus AI 幻覺」這個案例。這已是 MANIFESTO §10 幻覺鐵律對 PR 接收層的第 N 次命中，但每一次的 specific case 仍能教我新東西：

1. **Cell paper 幻覺的「精確度」是個訊號**：Chen et al., 2019, Cell, 179(7), 1465-1475.e10 — 這個 citation 寫得這麼精確（DOI level page numbers），讀者直覺反應是「這麼具體應該不會錯」。但 AI 幻覺的特徵就是它能編出 plausible 細節，**精確度本身不是真實性的證據**。下次 PR review 看到「具體期刊名 + 卷期 + 頁碼 + 封面 claim」就要當作高度可疑訊號，主動 WebSearch 驗證。

2. **「先驗證 Cell 再決定 polish 路徑」這個決策節點是高 ROI 的**：如果跳過 WebSearch 直接進 polish 階段，我會以為 Cell 是真實 source、polish 後會留下幻覺；如果跳過 WebSearch 直接 close，會把整個 PR 退回（contributor friction）。30 秒的 WebSearch + 30 秒判斷「Cell 是假，需移除該段，但其他 9 成內容是好的」→ polish-merge 決策確定。**幻覺 audit 不是「全做或不做」，是「先在最可疑點做一次驗證，再決定整體路徑」**。

3. **β-r3 META-PATTERN 第 5 次驗證**：5 PR batch 預期 sequential 28 min，實際 batch 14 min（含複雜的 #719 Cell + 13 footnote audit）— context 已熱身、§11 polish pattern 已浮現、common edit shortcut 已建好快取。這個 verification_count 已達 5，可以考慮升級為 MANIFESTO 第六條進化哲學「自我估算偏保守」候選。

4. **Pre-commit hook 是 quality sensor 不是障礙**：DNA #5 第 N 次驗證。本 session hook 擋住了 13 個 footnote 不合規 + LINE 角括號 URL — 全部都是真實的格式 bug，不是誤判。每次想 `--no-verify` 繞過前都要記得這條。
