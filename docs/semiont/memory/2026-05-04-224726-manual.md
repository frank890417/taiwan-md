走完一遍 REWRITE-PIPELINE 全程，**最有感的是工具基礎建設過去四週的密度**。今天上半段 article-health Phase 1-10 ship 完，11 個 plugin 從散落的 .sh + .py 整合到 SSOT 入口；EDITORIAL v5.6 補敘事層 4 章（核心矛盾 anchor / 具體性紀律 / 反向解釋編織 / 策展人筆記書寫紀律）；prose-health plugin 把原本 16 維 quality-scan + 3 tier manifesto-11 整合到 release-pr profile。下午 ship 的這些工具，晚上我跑王福瑞時 immediately 感受到：寫完一執行 `--profile=release-pr` 11 個 plugin 直接給我「哪裡 hard / 哪裡 warn」單一報告，不用記得跑哪幾個 .sh。link-target HARD 直接點到 L128 行哪個 wikilink 不存在；terminology HARD 直接抓「硬件」+「網絡」標出來；prose-health 把對位句 + 破折號 + AI metaphor + 空洞詞分 Tier 1/2/3 + score 累加。**這是工具讓 SOP 變成肌肉記憶的真實示範**——pipeline 不是因為熟了不用讀，pipeline 越熟工具越該替我接管 mechanical check，讓 attention 留給判斷層。

第二有感的是兩條 verbatim 引語策略決定。ARTFORUM 中文網是中國平台，王福瑞訪談原文是簡體中文。我 body prose 全用繁體，但簡體 verbatim 引語兩處（「艺术家因『一场大病过后』对『自身身体内的噪音』进行探索」+「噪音代表着没有规则，我可以直觉、随性地想像」）刻意保留簡體不轉繁體。原因：DNA #23 第 N 次驗證告訴我「verbatim」這個承諾的物理意義是「ctrl-F 在 source URL 可搜到」 — 我把簡體轉成繁體，那 ctrl-F 在 ARTFORUM 中文網就搜不到（簡體源 vs 繁體 transcription 失配）。同樣邏輯也讓我在顏峻引語遇到「硬件」中國用語 hard gate 時走 ellipsis 而不是改字。**verbatim 紀律的可操作邊界，是 ctrl-F 可搜性，不是「字面意思接近」**。今天這兩個 case 是這條邊界的實戰確認。

第三有感的是 NML peer ingestion 累積出來的 secondary source 質量。葉杏柔 2022 國藝會結案的「九〇年代噪聲作動的頻譜」六篇評論我這次直接 verbatim cite 三篇（〈How to NOISE〉/〈Before NOISE〉/〈異地是否有異響？〉），這是「Taiwan.md 的 NML peer ingestion 不只是內部 corpus 而已，它讓某類深度文章的 Stage 1 baseline 從『主 session 12 次 WebSearch』降到『主 session 5 次 WebSearch + 3 個 NML 本地原文 verbatim』」。peer 不是 source，但 peer 點出哪些 secondary source 值得當引用，省掉的是 attention overhead 不是 fact verification。
