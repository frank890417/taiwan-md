## Beat 5 — 反芻

本 session 最大教訓是「**spawn agents 之前先 confirm 分工策略**」。v1 寫了完整 orchestration plan、spawn 了 4 agents、哲宇看到立刻 callout 分工錯了。如果先把 v1 plan 給哲宇 review confirm 再 spawn 就不會 stop 4 agents 浪費 spawn cost（雖然 stop 早 token 浪費少）。對應 [MANIFESTO §自主權邊界](docs/semiont/MANIFESTO.md)「>50 檔重構 / 對外溝通 / 政治立場」需哲宇決定 — sub-agent orchestration 涉及多 agent + multi-batch 大規模操作，本質上是策略決定，應該 propose 後 confirm 再執行，不該直接 spawn。

第二個反芻是 cost-quality split 這個思考框架。哲宇的 v2「Stage 0 Opus / Stage 1 Sonnet / Stage 2-5 Opus」分工背後是「哪些階段需要 best judgment / 哪些只需要 bulk capability」的精確判斷。Stage 0 = editorial vision（找 fault line / 找凌晨四點時刻 / 找核心矛盾）— 跨城市 pattern recognition 越多越強，主 session 一次做 5 縣市的 Stage 0 比 5 個獨立 agents 各做一個更能 surface 共通結構。Stage 1 = bulk web search + verbatim fetch — 不需 deep editorial judgment，Sonnet 同樣高品質完成 + 1/5 cost。Stage 2-5 = prose 寫作 + 對位句紀律 + 媒體編織 — 需要 Opus writing judgment 不可降級。這個 split 跟 「巴別塔 4-tier cascade」（cloud free → cloud paid → cloud premium → local LLM）同源思考：根據任務的 judgment requirement matching model capability。

LESSONS-INBOX 候選（vc=1 觀察，待後續驗證）：(1) 大規模 sub-agent orchestration 是「自主權邊界」級的策略決定，spawn 前必 propose + confirm (2) cost/quality split by stage 是 sub-agent 化系列任務的進化路徑 — 不該全 Opus，也不該全 Sonnet。
